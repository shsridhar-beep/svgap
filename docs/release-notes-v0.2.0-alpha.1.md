# SV-Gap v0.2.0-alpha.1

This release turns the first research snapshot into an extensible handoff
framework for making research-to-production validity gaps explicit.

## Research scope

- Frames the primary result as an existential construct-validity claim rather
  than a population defect-rate estimate.
- Adds a compact research note and a claim hierarchy suitable for review-paper
  and benchmarking-critique use.
- Discloses that the 72 generation calls contain 69 unique normalized outputs.

## Evaluation contract

- Imports existing functional results with a cryptographic binding to the exact
  candidate sources.
- Discovers third-party structural checker backends through Python entry points.
- Tightens the report schema and validates reports before aggregation/export.
- Adds deterministic SARIF and dependency-free static HTML exports.

## Reset taskpack v0.2

- Preserves the eight reset-release behavior families without adding a new
  hazard category.
- Fixes the reset-timer specification ambiguity.
- Adds executable safe and unsafe references for every task.
- Calibration is tested before generation and the taskpack carries a canonical
  digest plus comparison protocol.

## Adoption and reproducibility

- Adds a pinned OSS CAD Suite container for Linux x86-64 and ARM64.
- Adds a reusable composite GitHub Action.
- Tests Python 3.11–3.13, verifies public artifacts in CI, builds release wheels,
  and exercises the container.
- Ships an end-to-end existing-benchmark integration example.

The built-in oracle remains a narrow reference implementation, not CDC/RDC
signoff. The public reset result remains a taskpack-conditional demonstration
count, not population prevalence or a model ranking.
